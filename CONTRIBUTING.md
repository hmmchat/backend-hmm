# Contributing to backend-hmm

## Workflow

- Create a feature branch from `main`
- Make your changes
- Open a Pull Request targeting `main`
- Address review feedback
- Merge after approval

## Development Setup

See [README.md](README.md) for prerequisites and setup. For frontend integration details, see [docs/for-frontend/](docs/for-frontend/).

## Code Style

- Follow existing patterns in the codebase
- Run `npm run lint` before committing
- Use TypeScript for all new code

## Pull Requests

- Keep PRs focused and reasonably sized
- Update documentation if you change APIs or behavior
- Ensure tests pass (run `./tests/run-all-tests.sh` for E2E)
